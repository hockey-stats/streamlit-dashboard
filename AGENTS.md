# AGENTS.md - Repository Guidelines

## Project Overview
Streamlit dashboard for fantasy baseball analytics, scraping data from Yahoo Fantasy API and pybaseball.

## Build/Run Commands

```bash
# Install dependencies (uses uv package manager)
uv sync

# Run the Streamlit dashboard
uv run streamlit run main.py

# Run data generation script
uv run python get_free_agent_summary.py

# Run batter stats script
uv run python get_detailed_batter_stats.py
```

## Testing
No test framework or tests currently exist. If adding tests:
- Use `pytest` as the test framework
- Place tests in a `tests/` directory mirroring source structure
- Run: `uv run pytest tests/`
- Run single test: `uv run pytest tests/test_file.py::test_name -v`

## Linting & Formatting
No linter/formatter is currently configured. Recommended additions:
- `uv run ruff check .` for linting
- `uv run ruff format .` for formatting
- Add `[tool.ruff]` section to pyproject.toml if adopted

## Code Style

### Imports
- Standard library first, then third-party, then local imports
- Use explicit imports (avoid `from module import *`)
- Group imports with blank line between groups

### Type Hints
- Use type hints on all function signatures and return types
- Import from `typing` module: `Dict`, `Tuple`, `Any`, `List`, `Optional`
- Example: `def get_park_factor(team: str) -> float:`

### Naming Conventions
- `snake_case` for functions, variables, and modules
- `UPPER_CASE` for constants (e.g., `THRESHOLDS`, `POSITIONS`)
- `CamelCase` for classes (none currently exist)

### Docstrings
- Use Google-style docstrings with `:param` and `:return` notation
- Include type in param/return lines: `:param str team: Description`
- Document all public functions and methods

### Error Handling
- Use `try/except` with specific exceptions (e.g., `KeyError`)
- Re-raise exceptions after logging when appropriate
- Use `raise ValueError("message")` for validation failures

### Data Processing
- Use `polars` (not pandas) for DataFrame operations
- Chain polars operations where readable
- Use `pl.col()` for column references

### Script Structure
- Use `if __name__ == '__main__':` guard for executable scripts
- Constants defined at module top, between comment markers
- Streamlit config (e.g., `st.set_page_config`) at module level

## File Structure
```
main.py                      # Streamlit dashboard entry point
get_free_agent_summary.py    # Data generation script (Yahoo API)
get_detailed_batter_stats.py # Batter statistics calculation
pyproject.toml               # Project configuration (uv)
requirements.txt             # Pinned dependencies
data/                        # Generated CSV data (gitignored)
```

## Sensitive Files
- `oauth.json` - Yahoo OAuth credentials (gitignored, use `oauth.json.gpg`)
- `.env` files with API tokens (gitignored)
- CSV data files are gitignored, generated by CI workflow

## CI/CD
GitHub Actions workflow (`.github/workflows/update_dashboard_data.yml`):
- Runs daily at 07:00 UTC and on pushes to `main`
- Generates CSV artifacts consumed by the dashboard
- Requires `YAHOO_OAUTH_DECRYPT_KEY` secret for GPG decryption
